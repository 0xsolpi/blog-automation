# blog-automation

블로그 자동화 운영을 위한 멀티 에이전트 오케스트레이션 저장소.

## 에이전트 구조

- Admin(사장님): 최종 관리자, 최종 승인권자
- 네레: 중간 관리자(오케스트레이터)
- 루피: 트렌드 수집
- 에이스: 쿠팡파트너스 검증
- 나미: 콘텐츠 작성
- 조로: 교정/품질/리스크 검수
- 로빈: 업로드/배포 보고

## 핵심 원칙

1. 사실 기반 작성(출처 포함)
2. 쿠팡파트너스 링크 없는 글은 배포 금지
3. 조로 검수 + 네레 승인 요청 + Admin 최종 승인 후 업로드
4. 모든 단계는 로그/산출물 파일로 남김

## 디렉토리

- `orchestration/`: 워크플로우/상태머신/승인게이트
- `agents/`: 에이전트별 역할/입출력 스펙
- `data/`: 실행 데이터(아이템 후보, 검증 결과, 초안, 최종본)
- `runs/`: 실행별 로그(YYYYMMDD-HHMM-runid)
- `ops/`: 운영 규칙, 장애 대응, 체크리스트

## 다음 단계

1. 워드프레스 업로드 방식 확정(XML-RPC/REST/API 플러그인)
2. 쿠팡파트너스 링크 수집/검증 방식 확정
3. 일일 스케줄(시작 시각/타임존/휴일 정책) 확정
4. OpenClaw 하위 에이전트 실행 스크립트 연결
